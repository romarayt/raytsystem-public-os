# Governance

raytsystem is currently a maintainer-led, pre-1.0 project.

Roma Rayt is the initial maintainer and has final responsibility for releases, security responses,
the roadmap, repository policy, and accepting changes. Decisions are made in public issues and pull
requests whenever disclosure is safe. Security incidents, private vulnerability reports, and any
discussion containing private corpus data remain private.

Contributors can propose changes through focused pull requests. Material changes to security
boundaries, canonical data, public contracts, or governance require documented rationale, tests,
updated documentation, and maintainer approval. Maintainer status may be expanded after sustained,
trusted contributions; no automatic tenure rule exists yet.

This model will be revisited before 1.0 or when more than one maintainer regularly shares release
and security duties.
