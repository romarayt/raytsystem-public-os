# Запуски (интерфейс)

## Что это

Маршрут `/runs` объединяет два журнала, переключаемых кнопками вверху: **«Live execution»** и **«Зафиксированные операции»** (`web/src/features/Runs.tsx`). Оба журнала доступны только для просмотра — повтор, продолжение и само выполнение управляются через CLI, а не через браузер.

## Когда использовать

- Посмотреть историю запусков и их состояния, не запуская ничего.
- Найти запуск по операции, задаче, исполнителю, провайдеру или ID.
- Отфильтровать записи по состоянию (успех, ошибка, карантин и т. п.).

## Журнал «Live execution»

Показывает очищенную историю execution runs (`web/src/features/ExecutionViews.tsx`). Для каждого запуска видны провайдер, короткие ID запуска и задачи, состояние, время старта, адаптер и учтённые токены. Сверху — сводка: активно, на проверке, требуют внимания, учтено токенов.

Новые запуски по умолчанию **выключены**. В `config/raytsystem.toml` флаг `runtime_execution_enabled=false`, поэтому раздел отображает баннер «Новые запуски отключены; сохранённая история доступна только для чтения». Если хранилище ещё не создано, раздел прямо сообщает: «Runtime выключен; execution store ещё не инициализирован», и подчёркивает, что GET-запрос не создаёт базу, workspace или graph snapshot.

Проекция намеренно **не** включает команду, рабочую директорию, environment и provider session — об этом говорит подпись под таблицей. Записи санитизированы.

## Журнал «Зафиксированные операции»

Показывает манифесты зафиксированных детерминированных запусков CLI (`LegacyRuns` в `web/src/features/Runs.tsx`). Таблица содержит операцию, состояние, время обновления, поколение и отпечаток манифеста (`manifest_sha256`). Доступны поиск и фильтр по состоянию: успех, ошибка, карантин.

Пустое состояние поясняет: история появится только после того, как детерминированный CLI-процесс зафиксирует манифест. Подпись под таблицей повторяет ключевое ограничение: «Запуски доступны только для просмотра. Повтор, продолжение и выполнение по-прежнему управляются через CLI».

## Ограничения и безопасность

- Оба журнала работают **только на чтение**; GET/HEAD ничего не создаёт и не пишет.
- Live execution по умолчанию отключён флагом `runtime_execution_enabled` и включается лишь явно.
- Записи санитизированы: команды, пути, окружение, секреты и provider session в проекцию не входят (redaction).
- Отпечатки манифеста и указатель поколения позволяют проверить целостность.
