# Интерфейс: обзор

## Что это

Веб-интерфейс raytsystem — это локальная панель управления вашим проверенным срезом знаний и работы. Он запускается командой `uv run raytsystem ui` и открывается только на петле `127.0.0.1`. Браузер показывает состояние, но сам не получает права выполнять команды, менять модели или публиковать знания.

Интерфейс состоит из четырёх постоянных частей:

- **Боковая навигация** слева — все разделы, сгруппированные по темам: «Пространство», «Оркестрация», «Реестр», «Доверие».
- **Верхняя панель** — название текущего раздела, кнопка палитры команд и переключатель темы.
- **Основная область** — содержимое выбранного раздела.
- **Инспектор** справа — компактные подробности задачи, запуска или узла графа. Agent
  и Skill открывают свои полные detail surfaces, чтобы метаданные и действия не перекрывали
  основной контент.

## Когда использовать

Начните с этого раздела, если вы впервые открыли raytsystem и хотите понять, где что находится, а также где заканчивается просмотр и начинаются действия, требующие подтверждения.

## Палитра команд (Cmd+K)

Палитра команд открывается сочетанием **⌘ K** (или **Ctrl+K**), а также кнопкой «Палитра команд» на верхней панели. Внутри Markdown-редактора **⌘/Ctrl+K** остаётся командой ссылки; глобальную палитру можно вызвать **⌘/Ctrl+Shift+P**.

## Read-only и изменяющие действия

Большинство экранов — только для чтения: они отображают проверенный срез и ничего не меняют. Изменяющих действий в интерфейсе немного, и все они локальны и ограничены своей плоскостью:

- **Создание задачи** и **перевод задачи** между статусами — в разделе «Задачи».
- **Обслуживание графа кода** (проверить, обновить изменённое, построить, пересобрать) — в разделе «Вселенная», линза «Код».
- **Предпросмотр и save editable Skill** — только для локального skill, который серверная policy
  разрешила редактировать; exact hashes, diff и audit обязательны.
- **Создание локальной копии read-only Skill** — отдельный preview/confirm поток, который
  не меняет исходник.
- **Создание, правка, rename и move документов** — только в явно writable root, с expected snapshot/hash, idempotency, audit и без delete.

Перед изменяющими действиями интерфейс явно предупреждает о границе: например, при создании задачи показывается плашка «Изменится только состояние задач. Канонические знания останутся нетронутыми».

## Ограничения и безопасность

- Интерфейс работает только локально (`127.0.0.1`), запросы GET/HEAD никогда не пишут.
- Браузер никогда не задаёт рабочий каталог, команду или переменные окружения.
- Если активен аварийный контур, вверху появляется плашка «Аварийный контур активен», и выполнение блокируется. Подробнее — [/security/emergency-controls](/security/emergency-controls).
- Выполнение агентов по умолчанию отключено: наличие профиля не означает, что агент запускается.
- Save Skill не запускает его, не выполняет tools и всегда оставляет новую revision в статусе
  `pending` до отдельной аттестации.
