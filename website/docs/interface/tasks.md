# Задачи (интерфейс)

## Что это

Раздел «Задачи» — операционный журнал работы. Это единственное место в интерфейсе, помимо обслуживания графа кода, где вы **меняете состояние**: создаёте задачи и переводите их между статусами. История задач неизменяема — переходы не переписывают прошлое, а добавляют новое состояние.

## Когда использовать

Открывайте «Задачи», когда нужно завести новую единицу работы, найти существующую задачу или продвинуть её по доске: из «Входящих» в «Запланировано», далее в «Готово», «В работе», «На проверке» и «Завершено».

## Основные элементы

По данным `web/src/features/Tasks.tsx`, раздел содержит:

- **Поле фильтра** — ищет по названию, описанию, тегам, статусу и приоритету. Фильтр только сужает отображение и ничего не меняет.
- **Значок поколения** — короткий идентификатор текущего поколения задач.
- **Кнопку «Новая задача»** — открывает форму создания.
- **Доску (канбан)** со столбцами: Входящие, Запланировано, Готово, В работе, На проверке, Заблокировано, Завершено, Отменено.

## Что меняет состояние

Изменяющих действий три, и все проходят через локальный API с проверкой ожидаемого поколения:

- **Создание задачи.** Кнопка «Новая задача» открывает форму с полями «Название», «Описание» (необязательно) и «Приоритет». Форма явно предупреждает: «Изменится только состояние задач. Канонические знания останутся нетронутыми».
- **Продвижение по доске.** Кнопка на карточке переводит задачу в следующий статус (например, «В работе» → «На проверке»).
- **Блокировка и отмена.** В меню карточки можно заблокировать задачу (с обязательной причиной) или отменить её (переход в финальное состояние). История при этом сохраняется.

## Связанный CLI

Те же операции доступны из терминала:

```bash
uv run raytsystem task list
uv run raytsystem task create
uv run raytsystem task transition
```

Для просмотра без изменений используйте `raytsystem task list`. Команды `task create` и `task transition` создают задачу и переводят её между статусами — это те же изменяющие операции, что и в интерфейсе.

## Ожидаемый результат

После создания или перехода задача появляется в нужном столбце доски, а вверху показывается уведомление вроде «Задача перемещена в …». Обновляются также сводки в «Центре управления».

## Ограничения и безопасность

- Изменяется **только состояние задач**; канонические знания не затрагиваются.
- Каждое изменение привязано к ожидаемому поколению задач: если срез уже изменился, интерфейс попросит обновить страницу и повторить действие.
- Блокировка требует конкретной причины; история задачи остаётся неизменяемой.

## Частые ошибки

- **Задача не переходит в нужный статус.** Проверьте, что переход допустим по жизненному циклу — см. [/tasks/lifecycle](/tasks/lifecycle) и [/troubleshooting/task-not-transitioning](/troubleshooting/task-not-transitioning).
