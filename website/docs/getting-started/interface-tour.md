# Экскурсия по интерфейсу

## Что это

raytsystem открывает единый веб-интерфейс командой `uv run raytsystem ui` по локальному
адресу `http://127.0.0.1:8765`. Интерфейс состоит из **одиннадцати разделов**, собранных в
левой панели в **четыре группы**. Эта страница — карта: куда идти за чем. Подробности
по каждому разделу — в статьях группы «Интерфейс».

## Четыре группы навигации

**Пространство**
- **Центр управления** (`command-center`) — состояние пространства и активная работа.
  Точка входа: сводка статусов рабочего пространства, работы, запусков и безопасности.
- **База знаний** (`handbook`) — публичная read-only документация raytsystem внутри UI.
- **Документы** (`documents`) — управляемые файлы и заметки текущего workspace.

**Оркестрация**
- **Задачи** (`tasks`) — операционный журнал без перезаписи истории. Создание задач и
  переходы статусов. См. [«Задачи»](/interface/tasks).
- **Вселенная** (`universe`) — граф знаний, работы и доказательств. Линзы **Орбита,
  Знания, Работа, Агенты, Доказательства** и линза **Код** (граф кода). Рендер на
  Sigma/WebGL, есть доступная текстовая альтернатива-список. См.
  [«Вселенная»](/interface/universe).
- **Запуски** (`runs`) — история зафиксированных операций.
  См. [«Запуски»](/interface/runs).

**Реестр**
- **Агенты** (`agents`) — один профиль на `agent_id`: определение и текущее execution-состояние.
- **Навыки** (`skills`) — полные безопасные Markdown-документы, permissions, hashes,
  история и policy-bound правка разрешённых local skills.
- **Контекст** (`context`) — разрешённые документы с инструкциями.

**Доверие**
- **Безопасность** (`safety`) — локальная граница и адаптеры.
  См. [«Безопасность»](/interface/safety).
- **Системы** (`systems`) — качество, политика и восстановление.
  См. [«Системы»](/interface/systems).

## Общие элементы экрана

- **Палитра команд** — кнопка «Палитра команд» в верхней панели или сочетание `⌘K` /
  `Ctrl+K`: быстрый переход между разделами и к объектам.
- **Инспектор** — правая панель с компактными деталями; Agent и Skill открывают полные detail
  surfaces с собственными вкладками.
- **Тема оформления** — переключатель тёмная → светлая → контрастная.
- **Индикатор «ЛОКАЛЬНО»** и статус выполнения внизу. По умолчанию отображается
  «выполнение отключено»: флаг `runtime_execution_enabled` выключен, интерфейс не
  запускает модель и не обращается к провайдеру.
- **Мобильная навигация** — нижняя панель с разделами «Главная», «Задачи», «Граф»,
  «Запуски» и кнопкой «Ещё» для остальных разделов.

## Куда идти за чем

- Посмотреть общий статус → **Центр управления**.
- Завести или продвинуть работу → **Задачи**.
- Понять связи в проекте и коде → **Вселенная** (линза «Код»).
- Проверить, что уже выполнялось → **Запуски**.
- Настроить и изучить агентов, навыки, контекст → группа **Реестр**.
- Проверить границы и адаптеры → **Безопасность**; политику и восстановление →
  **Системы**.

## Ограничения и безопасность

Интерфейс работает только на loopback-адресе `127.0.0.1`; несетевое связывание
отклоняется. Веб-интерфейс — это проверенный срез только для чтения плюс запись в
журнал задач и узкие hash-bound мутации local skills; выполнение моделей, shell, импорт и
публикация не имеют HTTP-эндпоинтов. Skill save не запускает текст и оставляет test status `pending`.
GET/HEAD никогда не пишут данные.
